## 1. 项目当前状态

LabCore 是一个面向课程学习与在线实验的学习平台，当前版本主要包含：

- 知识学习中心：学习卡片、分类筛选、学习详情、在线练习入口。
- Notebook 在线练习：基于 JupyterLite / Pyodide 打开卡片绑定的 `.ipynb` 模板。
- 后台内容管理：学习卡片管理、模板上传、模板恢复默认版本。
- 能力考核模块：学生端查看考题、填写文字答案、上传材料并提交；管理员端创建考核、查看提交、下载附件、评分或退回。
- 数据面板与用户管理：沿用原项目已有功能。

## 4. Notebook 与学习卡片交接

当前 Notebook 模板有两类目录容易混淆：

```text
data/learning-templates/
src/main/resources/static/experiments/
```

建议理解方式：

- `data/learning-templates/`：当前主要使用的学习卡片在线练习模板目录。后台上传模板后，一般保存为 `item-{cardId}/template.ipynb`。
- `src/main/resources/static/experiments/`：随 jar 打包的静态实验资源或备用资源，不建议和 `data/learning-templates/` 维护两套不同内容，容易造成“在线学习”和“在线练习”入口混乱。

本轮新增/导入过 5 个智能体学习卡片和 5 个 Notebook 模板。原始 JSON 数据位于：

```text
new_card/
```

相关导入预览 SQL：

```text
ops/import_learning_cards.preview.sql
```

JupyterLite 在线练习的现实限制：

- Pyodide 环境中的 `!pip install ...` 不等于服务器 Python 环境安装。
- 浏览器内 Notebook 是否能联网访问外部 API，取决于浏览器网络、目标 API CORS、Notebook 运行环境限制和用户自己的 API Key。
- 学生实验中涉及 API Key 的内容，应让学生自行填写，不要把实验用 Key 写入模板或仓库。

## 5. 能力考核模块交接

本轮新增了“能力考核”模块。命名上已经从早期“作业提交”逐步调整为“能力考核”，但部分代码、表名和接口仍使用 `assignment` 命名，这是当前实现约定。

### 5.1 学生端能力

学生端入口：

```text
左侧导航栏 -> 能力考核
```

学生可以：

- 查看已发布考核列表。
- 进入考核详情页阅读考题。
- 填写文字答案。
- 上传文档或视频。
- 提交考核。
- 查看得分、评语和退回状态。

学生详情页已调整为以“考题”为主：

- 顶部只保留简要状态信息。
- 考题区域作为主体内容。
- 右侧为文字答案、文档/视频上传和提交操作。

### 5.2 管理员端能力

管理员入口：

```text
右上角头像菜单 -> 考核管理
```

管理员可以：

- 新建考核。
- 编辑题目、分值、分类、截止时间。
- 发布、撤回或归档考核。
- 查看学生提交。
- 下载学生上传材料。
- 给分并填写评语。
- 退回重交。

“查看提交”页已调整为批改工作台视角：

- 左侧是较窄的学生提交索引。
- 右侧是主要区域，展示文字答案、附件和评分反馈。

### 5.5 上传文件规则

上传保存目录配置：

```properties
labcore.assignment.upload-dir=data/assignment-submissions
```

默认支持：

```text
文档：doc, docx, pdf, ppt, pptx, xls, xlsx, txt, zip, rar
视频：mp4, mov, avi, mkv, webm
```

上传大小配置：

```properties
spring.servlet.multipart.max-file-size=300MB
spring.servlet.multipart.max-request-size=330MB
```

部署到 Nginx 或其他代理后，需要同步调整代理层上传大小限制，否则后端配置即使正确，大文件仍可能在代理层被拒绝。

## 6. 数据库脚本说明

数据库脚本统一放在：

```text

```

主要脚本：

```text

```

完整初始化/安全补丁脚本，适合新环境或需要一键建完整结构时使用。应包含管理员初始账号处理、学习卡片基础结构、能力考核结构和智能体卡片数据。

```text

```

能力考核模块独立建表脚本，适合已有 `labcore` 数据库升级时执行。

```text
assignment_answer_text_migration.sql
```

为提交记录增加 `answer_text` 字段的迁移脚本，用于支持学生直接填写文字答案。

```text
seed_assessment_demo.sql
```

本地演示数据脚本，只用于页面验收或本地测试，不建议导入正式生产环境。

```text

```

干净建表脚本，不包含完整业务种子数据。

注意：

- 正式部署前请先备份数据库。
- 服务器执行 SQL 时使用服务器内置 MySQL 客户端路径。
- Demo 脚本不要导入生产库。

## 8. 部署交接

堡垒机部署原则：

- 尽量不在服务器上在线安装依赖。
- 本地构建 jar、前端 dist、SQL 和部署脚本后打包上传。
- 服务器上使用已有 Java、MySQL 和 Web 运维能力完成替换。

本轮曾生成过部署相关文件：

```text
ops/deploy_labcore_assessment_update.sh
ops/deploy_assessment_db.sh
ops/labcore-assessment-update-20260615.zip
```

其中 zip 属于部署包产物，不建议提交 GitHub；脚本和说明可以保留。

常规部署思路：

1. 本地构建后端 jar。
2. 本地构建前端 dist。
3. 上传部署包到堡垒机。
4. 在服务器备份旧版本和数据库。
5. 执行能力考核相关 SQL。
6. 替换后端、前端产物。
7. 重启服务。
8. 验证 `https://labcore.henu.edu.cn/` 的登录、学习卡片、在线练习、能力考核、考核管理。

## 9. 已知注意事项

### 9.1 认证与静态资源

不要为了让 Notebook 静态资源加载而粗暴放开所有认证。之前讨论过，正确方向是基于原认证逻辑修复资源访问路径，而不是扩大未登录访问范围。

### 9.2 错误提示

前端曾出现英文报错，例如注册用户名重复只显示 `Request failed with status code 400`。当前已做过中文提示改进，后续新增接口也应继续返回清晰中文错误。

### 9.3 下载文件名

学生上传 PDF 后下载曾出现 UUID 文件名问题。当前下载接口应通过 `Content-Disposition` 返回原始文件名，前端解析并保留原名。

### 9.4 本地上传材料

`data/assignment-submissions/` 是本地运行产生的学生上传材料，不应提交仓库。

### 9.5 部署包目录

`ops/labcore-assessment-update-*` 是解压后的部署包目录，不应提交仓库。
