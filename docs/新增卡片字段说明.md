# 新增卡片字段说明

下面这张表对应当前学习平台“新增学习卡片”时提交的主要参数。

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `title` | `String` | 卡片标题，必填。学生端会把它作为知识学习页面的主标题。 |
| `summary` | `String` | 卡片摘要，用来简要介绍这张卡片的学习内容。 |
| `category` | `String` | 卡片分类，例如“现代密码学”“区块链安全专题”。 |
| `difficulty` | `String` | 难度等级，通常填写“简单”“中等”“困难”。 |
| `duration` | `String` | 预计学习时长，例如“2小时”“3小时”。 |
| `prerequisites` | `String` | 前置要求，说明学习这张卡片前建议掌握哪些基础知识。 |
| `objectives` | `List<String>` | 学习目标列表。前端一般按“每行一个目标”录入，提交时转成字符串数组。 |
| `features` | `List<LearningFeature>` | 特色亮点列表，每一项包含亮点标题和亮点描述。 |
| `features[].title` | `String` | 单个亮点的标题。 |
| `features[].description` | `String` | 单个亮点的说明文字。 |
| `steps` | `List<StepInput>` | 学习步骤列表，决定学生端“学习步骤”区域显示的内容和顺序。 |
| `steps[].stepNo` | `Integer` | 步骤序号，例如 1、2、3。 |
| `steps[].title` | `String` | 单个学习步骤的标题。 |
| `steps[].description` | `String` | 单个学习步骤的正文说明。 |
| `steps[].tip` | `String` | 单个学习步骤的提示信息。 |
| `steps[].code` | `String` | 单个学习步骤对应的代码示例或代码片段。 |

## 补充说明

- 当前后端保存新增卡片时，真正强制必填的是 `title`。
- 如果要让这张卡片真正可用，建议至少补齐：
  - `title`
  - `summary`
  - `category`
  - `difficulty`
  - `duration`
  - `objectives`
  - `steps`
- 在线练习模板 `.ipynb` 不在这次新增卡片请求体里提交，而是保存卡片后通过“上传模板”单独上传。
